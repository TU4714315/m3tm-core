# M3TM Core Architecture

## الهدف

تصميم نواة عملية لجمع المحتوى العام، تصنيفه، تلخيصه، وإرسال التنبيهات إلى قناة متابعة.

## المسار التنفيذي

1. مصادر عامة: RSS وواجهات رسمية ومحتوى منشور للعامة.
2. n8n: إدارة التدفقات والجدولة.
3. LLM: التصنيف والتلخيص واستخراج الكيانات.
4. تخزين: PostgreSQL لاحقا للنتائج والسجلات.
5. تنبيهات: Telegram عند تحقق شروط الأهمية.

## الطبقات

### 1. Ingestion Layer
تجميع البيانات من مصادر عامة وقانونية.

### 2. Processing Layer
تنظيف النصوص وإزالة التكرار واكتشاف اللغة.

### 3. Analysis Layer
تصنيف الموضوع، استخراج الكيانات، قياس الأهمية، وتوليد ملخص.

### 4. Alert Layer
إرسال إشعارات مختصرة فقط عند تحقق شروط التنبيه.

### 5. Archive Layer
حفظ النتائج والسجلات للبحث والمراجعة.

## مبادئ التشغيل

- لا يتم جمع بيانات خاصة.
- لا يتم تجاوز قيود المواقع أو الحماية.
- كل نتيجة يجب أن تحمل رابط المصدر ووقت الجلب.
- أي نتيجة منخفضة الثقة يتم وسمها بوضوح.
